# Контракт поведения Alarm Overlay

Документ фиксирует эталонное поведение и текущее реализованное состояние после шагов B-D.

## 1. Логика потери датчиков

- Потеря `T1`, `T2`, `T3`, `dT`, `P` не останавливает процесс: связанные реле остаются в текущем положении, команда управления считается нейтральной и не формирует `forbid`.
- Поведение `L` и `F` не меняется: соответствующая защита останавливает связанный канал через `forbid`.
- Рабочий диапазон температуры для целей контракта: `-50..+125 C`.
- Рабочий диапазон давления для целей контракта: `800..1300 гПа`.
- Приоритет ошибки чтения принадлежит библиотеке датчика: аппаратная ошибка чтения трактуется как ошибка датчика независимо от текущего значения.
- Выход измерения за рабочий диапазон трактуется как ошибка датчика.
- Реализация приведена к диапазонам постановки: код использует диапазон температуры `-50..125 C` и диапазон давления `800..1300 гПа`.

## 2. Надпись «Ошибка датчика» (active / sticky)

- `sensorErrorActive` описывает текущую физическую неисправность датчика.
- `sensorErrorActive` ставится при fault и снимается автоматически после стабильного healthy-состояния и гистерезиса `SENSOR_HEALTHY_HYSTERESIS_MS`.
- `sensorErrorSticky` описывает только операторскую отметку на кнопке датчика.
- `sensorErrorSticky` ставится при fault и не снимается квитированием тревог.
- После физического восстановления датчика показания и участие в автоматике возвращаются автоматически, а датчик исчезает из окна ошибок.
- Надпись `Ошибка датчика` на кнопке остаётся видимой, пока `sensorErrorSticky == true`.
- `sensorErrorSticky` снимается только вручную через цикл `выключить -> включить` в конфигурации датчика.
- Снятие `sensorErrorSticky` выполняется через `applyOperatorResetCycle` и вызов из `_handleSensorConfig`, без зависимости от текущего healthy-состояния.

## 3. Всплывающее окно (overlay)

- Всплывающее окно отображается поверх всего экрана.
- Всплывающее окно носит только информативный характер.
- Всплывающее окно содержит только текстовый список сработавших тревог и отдельный текстовый список потерянных датчиков.
- Во всплывающем окне не допускаются кнопки `Квитировать`, `Свернуть`, badge и любые другие интерактивные элементы.
- После квитирования всплывающее окно гаснет вместе со звуковой сигнализацией.
- После квитирования надпись `Ошибка датчика` на кнопке датчика остаётся, если `sensorErrorSticky == true`.

## 4. Квитирование

- Квитирование выполняется штатной кнопкой квитирования в основном интерфейсе и/или аппаратной кнопкой `ACK`.
- `POST /api/v1/ack` сбрасывает активные тревоги, связанные со звуком и всплывающим окном.
- `POST /api/v1/ack` не снимает `sensorErrorSticky`.
- После `ack` всплывающее окно закрывается, если не осталось неквитированных тревог.

## Дефекты

- Дефект 1 исправлен на шагах B/C: штатная кнопка `Квитировать` использует свежий ответ `POST /api/v1/ack`, после квитирования гасит звук и закрывает всплывающее окно без ожидания следующего `loadState()`.
- Дефект 2 исправлен на шаге C: из всплывающего окна удалены элементы `Квитировать`, `Свернуть` и badge; окно оставлено только информативным.

## Открытые вопросы

- Закрыто: диапазоны `T` и `P` приведены к постановке `-50..125` и `800..1300`.
- Закрыто: `ack` не трогает `sensorErrorSticky`; снятие sticky-метки остаётся только через `applyOperatorResetCycle` из конфигурации датчика.
