# FIX NOTES 1.6.44

## Что исправлено

- На главном экране добавлено отображение текстовых причин активного квитирования, если кнопка `Квитирование` активна.
- Список причин использует уже существующие тексты тревог и ошибок, переданные backend'ом через `activeAlarmReasons`.
- Для нескольких одновременных причин они показываются списком в блоке `Активные тревоги`.

## Какие причины теперь доходят до UI

- `Потеря датчика T1`
- `Потеря датчика T2`
- `Потеря датчика T3`
- `Ошибка dT: невозможно вычислить значение`
- `Потеря датчика P`
- `Потеря датчика F`
- `Авария уровня: цепь L разомкнута`

## Что не менялось

- Логика датчиков.
- Логика реле.
- Логика STOP.
- Логика квитирования.
- Условия возникновения и сброса тревог.
- Тексты уже существующих записей журнала.

## Техническая заметка

- `/api/v1/state` и `/api/v1/ack` теперь могут отдавать массив `activeAlarmReasons` для показа уже активных причин квитирования в UI.
